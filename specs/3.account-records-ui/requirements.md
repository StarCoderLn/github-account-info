# 账户记录表单页 — 需求规格

## 需求版本

| 日期       | 版本 | 说明                                                         |
| ---------- | ---- | ------------------------------------------------------------ |
| 2026-06-27 | v1   | 初始需求：`/accounts` 单页 CRUD 表格 + 弹窗表单              |
| 2026-06-28 | v2   | 重构为 Token 管理 + 账号信息双页，UI 交互大幅调整            |

## 概述（v2）

将账户管理拆分为两个独立页面：

- **Token 管理页（`/`）**：添加并管理 GitHub PAT，以账号卡片形式展示，点击进入账号信息页。
- **账号信息页（`/profile`）**：展示并编辑账号详情，支持从 DB 或 GitHub 加载数据，保存入库。

## 项目信息

- 项目名: github-account-info
- 架构类型: pnpm + Turborepo monorepo（apps/web = React + TanStack Router + Tailwind）

## 用户故事

- 作为用户，我想添加多个 GitHub PAT 并以卡片形式查看对应账号信息，点击卡片进入编辑页。
- 作为用户，我想在账号信息页查看/编辑 GitHub 账号数据，并保存到数据库。
- 作为用户，我希望已保存过的账号下次进入时直接从数据库加载，而不是每次重新拉取 GitHub。
- 作为用户，我可以随时手动触发「从 GitHub 刷新」覆盖本地数据。

## 功能需求

### Token 管理页（`/`）

1. [F-001] 添加 Token 表单：填写名称 + PAT，验证通过后保存到 localStorage 并以卡片展示。
2. [F-002] Token 验证：添加时调用 `github.getAccount` 接口，失败则提示错误，不保存。
3. [F-003] 账号卡片：展示头像、用户名（login）、displayName、公开仓库数、关注者数、正在关注数。
4. [F-004] 点击卡片：将该 Token 设为当前选中，导航至 `/profile` 页。
5. [F-005] 删除 Token：卡片右上角删除按钮，从 localStorage 移除。

### 账号信息页（`/profile`）

6. [F-006] 数据加载优先级：DB 有记录 → 从 DB 加载；DB 无记录 → 用选中 Token 从 GitHub 拉取。
7. [F-007] 显示数据来源标注：「数据来源：数据库」或「数据来源：GitHub」。
8. [F-008] 可编辑字段：姓名、简介、公司、地址、邮箱、博客 URL、Twitter 用户名。
9. [F-009] 只读展示：头像、login、GitHub ID、公开仓库数、关注者数、正在关注数。
10. [F-010] 保存逻辑：DB 有记录则更新（`account.update`），无记录则新增（`account.create`）。
11. [F-011] 「从 GitHub 刷新」按钮：手动拉取最新数据覆盖表单，不自动保存。
12. [F-012] 未选中 Token 时显示引导提示，跳转回 Token 管理页。

## 非功能需求

- **数据持久化**：Token 完整值存于 localStorage，供自动拉取和刷新使用（个人本地部署场景可接受）。
- **安全说明**：localStorage 存储 PAT 在个人本地使用场景风险可控；如需多人部署，应改为服务端 session 存储。
- **体验**：loading 状态区分「加载中」与「正在从 GitHub 拉取」；写操作有 toast 成功/失败反馈。

## 验收标准

- [x] [AC-001] Token 管理页：输入有效 PAT 点「添加」后出现账号卡片，卡片含头像和统计数据。
- [x] [AC-002] Token 管理页：点击卡片跳转到账号信息页，并自动加载对应账号数据。
- [x] [AC-003] 账号信息页：DB 有记录时从数据库加载，来源标注显示「数据库」。
- [x] [AC-004] 账号信息页：DB 无记录时从 GitHub 拉取，来源标注显示「GitHub」。
- [x] [AC-005] 编辑后点「保存到数据库」，再次进入页面加载的是已修改的数据库版本。
- [x] [AC-006] 「从 GitHub 刷新」覆盖表单但不自动入库，需再次点保存才持久化。
- [x] [AC-007] 接口报错（无效 token、唯一冲突等）通过 toast 给出可读提示。
